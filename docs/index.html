Workflow

How Echo works.

From your voice to refined text in four steps — one keypress.

  1. 1

    Record

    Press ⌘⇧V or hold fn and start talking.

  2. 2

    Transcribe

    Speech becomes text via local Whisper, Groq, or macOS Speech.

  3. 3

    Refine

    An LLM cleans it up with context from your apps and vocabulary.

  4. 4

    Insert

    The final text is typed at your cursor — right where you were.

// point Echo at your repo
scan ~/code/echo

// now it knows your terms
"refactor the run_pipeline fn in lib.rs"
"check transcribeAudio in pipeline.ts"

✓ transcribed exactly — no typos

For developers

Dictate code and jargon — accurately.

Point Echo at a codebase and it learns your class names, function names, and project vocabulary. Technical terms get biased into the transcriber before the LLM runs, so run_pipeline never becomes "run pipe line."

  • Codebase scanning for project terminology
  • Custom vocabulary + auto-learned corrections
  • Caret-aware sentence continuation
  • Great context for Claude CLI & API refinement
